# 🍕 Pizza Sales Analysis Using SQL

## 📖 Project Overview

This project focuses on analyzing pizza sales data using SQL to uncover valuable business insights related to revenue, customer behavior, product performance, and sales trends. The analysis was performed using MySQL on a relational database consisting of order, product, and category information.

The goal of this project is to transform raw transactional data into actionable insights that can help businesses improve inventory management, pricing strategies, marketing campaigns, and operational efficiency.

---

## 🎯 Objectives

- Analyze overall sales performance.
- Calculate total revenue and average order value.
- Identify best-selling and least-selling pizzas.
- Evaluate category-wise and size-wise sales performance.
- Discover peak ordering hours and busiest days.
- Analyze daily and monthly revenue trends.
- Generate business insights through SQL queries.

---

## 🗂 Dataset Information

The project uses four tables:

### 1. Orders
Contains order date and time information.

### 2. Order Details
Contains transaction-level order records and quantities sold.

### 3. Pizzas
Contains pizza size and pricing information.

### 4. Pizza Types Fixedd
Contains pizza names, categories, and ingredients.

## 📈 Key Insights

- Generated over **$817K** in total revenue.
- Processed more than **21,000 customer orders**.
- Sold approximately **49,000 pizzas**.
- Large-sized pizzas generated the highest revenue.
- Classic category emerged as a top-performing category.
- Peak order activity occurred during lunch and evening hours.
- Friday recorded the highest order volume.
- A small number of pizza varieties contributed significantly to total revenue.
